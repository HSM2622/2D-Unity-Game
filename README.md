# 🚀 2D Warrior

## 📄 プロジェクトの概要 (Project Overview)

障害物とモンスターが混在するステージを通過し、ボスを倒して勝利を目指す **2D横スクロールアクションアドベンチャーゲーム**。

## 🛠️ プロジェクトの状態 (Project Status)

**デモ版 (Demo Version)**
* **実装内容:** 障害物ステージ 2つ、ボスステージ 1つを実装済み。
* **目的:** ポートフォリオ展示用として、コアな戦闘システムとレベルデザインの能力を示すことを目的としました。

---

## 🖥️ インストールと実行方法 (Setup & Run)

ローカル環境でプロジェクトを開くための手順です。

### 1. 動作環境

| 項目 (Item) | 詳細 (Detail) |
| :--- | :--- |
| **Unity バージョン** | **Unity 2021.3.7f1** |
| **動作確認 OS** | Windows / macOS |

### 2. 実行手順

1.  GitHubからリポジトリをクローンします。
    ```bash
    git clone [ここにGitHubリポジトリのURLを挿入]
    ```
2.  Unity Hubを開き、プロジェクトを登録します。
3.  `Assets/Scenes/MainScene.unity` ファイルをダブルクリックして開きます。
4.  Unityエディタの **'Play'** ボタンを押してゲームを開始します。

## ⚙️ 技術スタックとアピールポイント (Technology Stack & Appeal)

プロジェクトの実装に使用した主な技術と、開発を通じて証明されたスキルです。

### 1. 使用技術

* **Unity Engine:** **2021.3.7f1**
* **言語:** C#
* **コアシステム:** **リアルタイム戦闘**、2Dプラットフォーム制御、簡易的な敵AI

### 2. 技術アピールポイント (技術的な強み)

#### 🔸 デザインパターンの活用
* **インターフェース (`IObject`) の活用**： プレイヤー、敵 (`Skeleton`, `Enemy`, `Bringer`), トラップなど全てのゲームオブジェクトに対して共通の振る舞い（`OnDamaged`, `Die`, `OnAttack`）を定義し、**疎結合**なコード構造を実現しました。これにより、オブジェクトの拡張と保守が容易になっています。

#### 🔸 リアルタイム戦闘と状態管理
* **リアルタイム戦闘の実装**： プレイヤーの**無敵時間 (`OffDamaged`)**、攻撃クールタイム、敵の攻撃判定（**Physics2D.OverlapBoxAll** を使用）など、アクションに必要な状態管理ロジックを実装しました。
* **敵のAIロジック**：
    * `Skeleton` / `Bringer` (ボス) では、**感知範囲 (`detect`)**に基づきプレイヤーを自動追尾・攻撃するロジックを実装しました。
    * `Enemy` (一般敵) では、**ランダムな移動パターン**とプラットフォーム境界検知（**Raycast**）を実装し、ステージから落下하지 않도록制御しています。

    * **UI:** ユーザーインターフェース関連のプレハブおよび画像。
